Integração: Desacoplamento e Mensagens

Manter arquiteturas flexíveis e resilientes a picos de tráfego usando Pub/Sub e Filas.

Serviço AWS Definição Teórica e Aplicações Práticas
Amazon SQS (Simple Queue Service)

Serviço de mensagens em fila (Pull). Fundamental para desacoplar componentes (decoupling) e amortecer picos de carga. O SQS FIFO garante que a ordem é mantida exatamente 1 vez.

Casos Práticos do Simulado:
  • O Amazon SQS (Simple Queue Service) é um serviço de enfileiramento de mensagens totalmente gerido e altamente fiável que permite desacoplar componentes de software.
  • O Amazon SQS FIFO Queue garante a entrega de mensagens exatamente uma vez (Exactly-Once processing) e a ordem estrita First-In, First-Out, possuindo suporte nativo a deduplicação de mensagens basea...
  • O SQS FIFO (First-In, First-Out) é projetado especificamente para garantir que as mensagens sejam processadas exatamente uma vez e mantendo a ordenação estrita.
Amazon SNS (Simple Notification Service)

Serviço de notificações Pub/Sub (Push). Envia mensagens para múltiplos assinantes simultaneamente (Emails, SMS, ou outras filas SQS "fan-out").

Casos Práticos do Simulado:
  • O AWS Budgets permite configurar orçamentos personalizados para acompanhar custos ou utilização na AWS.
AWS Step Functions

Máquina de estados visual que orquestra componentes de microsserviços. Ideal para gerir o fluxo longo e condicional de várias funções Lambda.

Casos Práticos do Simulado:
  • O AWS Step Functions é um serviço de orquestração visual serverless de fluxos de trabalho baseados em máquinas de estado.
Amazon EventBridge

Autocarro de eventos (Event Bus). Substituiu o CloudWatch Events. Despoleta reações (ex: acionar um Lambda) sempre que um evento na sua conta ocorre.

Casos Práticos do Simulado:
  • O Amazon EventBridge é um barramento de eventos serverless que facilita a conexão de aplicações usando dados de recursos AWS, de aplicações SaaS e aplicações próprias do utilizador.
Amazon Kinesis Data Streams

Serviço projetado especificamente para ingerir um volume maciço de dados em fluxo (streaming) em tempo real (ex: logs de IoT ou de clickstream).

Casos Práticos do Simulado:
  • O Amazon Kinesis Data Streams é projetado especificamente para ingerir e processar fluxos de dados massivos em tempo real em larga escala.
