Computação: EC2, Spot e Autoscaling

A base da computação na AWS, lidando com máquinas virtuais e a sua escalabilidade.

Serviço AWS Definição Teórica e Aplicações Práticas
Amazon EC2

Servidores virtuais. Deve ser usado quando necessita de acesso total ao Sistema Operativo ou para software legacy.

Casos Práticos do Simulado:
  • O AWS Fargate Spot é extremamente económico para cargas de trabalho baseadas em contentores tolerantes a falhas (até 70% de desconto face ao preço Fargate normal), sem o overhead de gerir grupos de...
  • A hibernação do EC2 guarda o estado da memória RAM no volume EBS raiz.
  • O AWS Nitro Enclaves permite criar ambientes de computação altamente isolados (enclaves) dentro de instâncias EC2.
  • O RDS Multi-AZ clássico fornece replicação síncrona de dados para uma instância passiva noutra Zona de Disponibilidade da mesma região, oferecendo RPO de 0 segundos (perda zero) e failover automáti...
  • O Systems Manager Session Manager permite sessões seguras e controladas a instâncias EC2 através do browser ou AWS CLI, sem a necessidade de chaves SSH, IPs públicos ou portas SSH expostas na inter...
  • As instâncias EC2 Spot aproveitam a capacidade de computação sobressalente da AWS com descontos maciços de até 90% em comparação com o preço sob demanda.
  • O RDS Multi-AZ fornece replicação síncrona automática para uma instância passiva noutra AZ, garantindo RPO = 0 (sem perda) e failover automático rápido de sub-minuto.
Spot Instances

Aproveita a capacidade de computação sobressalente da AWS com enormes descontos. Podem ser interrompidas. IDEAL PARA: Workloads tolerantes a falhas e flexíveis.

Casos Práticos do Simulado:
  • O AWS Fargate Spot é extremamente económico para cargas de trabalho baseadas em contentores tolerantes a falhas (até 70% de desconto face ao preço Fargate normal), sem o overhead de gerir grupos de...
  • A hibernação do EC2 guarda o estado da memória RAM no volume EBS raiz.
  • As instâncias EC2 Spot aproveitam a capacidade de computação sobressalente da AWS com descontos maciços de até 90% em comparação com o preço sob demanda.
  • O Fargate Spot fornece capacidade serverless com descontos massivos de até 70% comparado ao Fargate padrão, perfeito para tarefas tolerantes a falhas rápidas.
Auto Scaling Groups (ASG)

Adiciona ou remove instâncias EC2 automaticamente com base em métricas. Essencial para Alta Disponibilidade (espalhar por múltiplas AZs).

Teoria fundamental para o exame. Não testado diretamente nos seus simulados.
Application Load Balancer (ALB)

Distribui tráfego HTTP/HTTPS (Layer 7). Pode rotear com base no caminho HTTP (ex: /images vai para um grupo) e interagir com Lambdas.

Casos Práticos do Simulado:
  • O Step Scaling responde a alarmes do CloudWatch de forma incremental (adicionando mais instâncias dependendo da diferença entre a métrica atual e o limite do alarme).
  • O Application Load Balancer suporta a integração nativa com funções Lambda e permite o roteamento de caminhos HTTP (por exemplo, /api/users para Lambda-Users, /api/orders para Lambda-Orders).
  • O AWS WAF filtra requisições na camada 7 (HTTP/HTTPS) contra injeções SQL e Cross-Site Scripting (XSS).
  • O AWS WAF protege as camadas de aplicação contra SQLi e XSS, enquanto o CloudFront e o Shield Advanced defendem de forma integrada contra ataques DDoS volumétricos (camadas 3/4/7).
Network Load Balancer (NLB)

Distribui tráfego TCP/UDP (Layer 4) para altíssima performance, com milhões de pedidos por segundo a latências ultra-baixas.

Teoria fundamental para o exame. Não testado diretamente nos seus simulados.
